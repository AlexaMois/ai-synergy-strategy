| **Кэширование (1597 КБ)** | Cache headers настраиваются на уровне хостинга Lovable, не в коде проекта. |

### Что МОЖНО исправить (4 задачи)

#### 1. Forced reflow (принудительная перерисовка) — 160 мс
Основной виновник — `ui-components` (158 мс), скорее всего Radix UI компонент, запрашивающий `offsetWidth` при рендере. Нужно проверить, какой именно компонент и можно ли отложить его рендер.

#### 2. Неиспользуемый CSS (15 КБ)
74% CSS не используется на главной. Можно вынести стили специфичных страниц (print, snowfall, marquee, animations) в отдельные CSS-модули, загружаемые лениво.

#### 3. Render-blocking CSS (50 мс)
Единственный блокирующий ресурс — основной CSS. Можно инлайнить критический CSS и загружать остальное асинхронно.

#### 4. Unsized images (логотип без width/height)
Логотип в навигации (`logo-horizontal.png`) не имеет атрибутов `width` и `height`, что вызывает layout shift (CLS).

---

### Рекомендация

Из этих 4-х реально стоит исправить только **пункт 4** (добавить `width`/`height` логотипу) — это простое изменение с реальным эффектом на CLS.

Пункты 1-3 дадут минимальный прирост (суммарно ~50-100 мс на десктопе) при значительной сложности реализации. Score 81 — это хороший результат для SPA с аналитикой.
